選擇技術方案

三個方向，各有取捨

A

React Native + Expo ⭐ 推薦

優點

  • TypeScript / JavaScript，學習曲線低
  • Expo SDK 內建 GPS、通知、相機 API
  • 一份程式碼跑 iOS + Android
  • 生態系成熟，社群龐大

缺點

  • 效能略遜於 Flutter（一般應用感受不到）
  • 背景定位需 expo-task-manager，設定稍複雜
主要套件： Expo SDK 52、React Native 0.76、expo-location、expo-notifications、expo-sqlite、react-navigation v7
B

Flutter

優點

  • 效能接近原生，UI 一致性高
  • 自帶豐富 Widget，視覺效果精美
  • Dart 語言型別系統嚴格

缺點

  • 需學 Dart 語言
  • App 包體積偏大
  • 背景定位 plugin 生態不如 Expo 完整
主要套件： Flutter 3.27、geolocator、flutter_local_notifications、sqflite、go_router
C

原生 (Swift + Kotlin)

優點

  • 最佳效能與系統整合
  • 背景定位功能最完整

缺點

  • 需維護兩份程式碼（iOS / Android）
  • 開發時間 2 倍以上
